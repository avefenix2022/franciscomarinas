CSS Multi-column Layout

El módulo CSS Multi-column Layout proporciona una solución a cómo organizar el contenido para que ocupe múltiples contenedores verticales.

El módulo CSS Multi-column Layout proporciona una solución a uno de los problemas fundamentales de diseño que han enfrentado a los diseñadores web desde los primeros días de la Web: cómo organizar el contenido para que ocupe múltiples contenedores verticales, como un periódico o una revista.
Con el paso de los años, diseñadores web ingeniosos han desarrollado una variedad de soluciones para crear diseños de varias columnas. Inicialmente esto implicaba usar tablas para alinear columnas de contenido como si cada elemento fuera un valor en una celda. Más recientemente, combinaciones inteligentes de elementos flotantes, elementos de compensación y JavaScript ocasionalmente han proporcionado una solución más semántica, pero ninguno de estos enfoques es particularmente adecuado para crear diseños de varias columnas. El nuevo módulo CSS3 supera muchos de los problemas asociados con la generación de dichos diseños, manejando automáticamente el flujo y la distribución del contenido; Y el soporte del navegador ya es bastante bueno, por lo que es relativamente seguro empezar a usar este módulo hoy!

El módulo de columnas múltiples amplía el modelo de caja CSS existente. Los navegadores que no entienden las propiedades de diseño de varias columnas los ignoran, convirtiendo el área de contenido en una sola columna en lugar de varias columnas.

Sintaxis y Estructura

El módulo CSS Multi-column Layout proporciona una solución a cómo organizar el contenido para que ocupe múltiples contenedores verticales.

El módulo CSS Multi-column Layout amplía el modelo de caja CSS existente con un total de diez propiedades adicionales.
Cada uno de ellos proporciona control sobre un aspecto de la forma en que el contenido se procesa en columnas dentro del contenedor. Debido a que todas las propiedades afectan a un único contenedor, el módulo es muy fácil de entender y, lo más importante, fácil de implementar desde el punto de vista del navegador.

Conceptos Básicos

El módulo CSS3 Multi-column Layout introduce diez nuevas propiedades que se pueden aplicar a elementos de nivel de bloque. Además de las nuevas propiedades que controlan el número de columnas, distribución, ancho y divisiones, algunas nuevas palabras clave están disponibles para ayudar a controlar cómo se rompe el contenido de las columnas.

Crear un diseño de varias columnas es muy sencillo con el módulo de múltiples columnas. No hay propiedades innecesariamente complicadas de aprender; Y porque todo sucede en un solo elemento, no hay necesidad de preocuparse acerca de cómo el contenido se envolverá fuera de su contenedor

Con sólo las diez propiedades, puede hacer lo siguiente:

  • Convertir un elemento de una sola columna existente en un elemento de varias columnas
  • Incluir contenido automáticamente entre columnas
  • Controlar cómo las columnas llenan el espacio disponible, ya sea expandiéndolo para llenar el contenedor o bien siempre se adhiere a un tamaño preestablecido (que es útil para crear diseños responsivos)
  • Definir los bordes que aparecen entre las columnas y el ancho del canal
  • Renderizar el contenido para que abarque varias columnas para romper el diseño columnar cuando se desee

Vale la pena destacar el tercer elemento en la lista de cosas CSS Multi-column Layout le permite hacer. Es un módulo de diseño ideal para usar si desea orientar y paginar contenido para múltiples configuraciones de dispositivos diferentes.
La naturaleza sensible de las columnas CSS no se limita al texto. Las imágenes también se adhieren a la disposición de la columna, por lo que mediante técnicas de imagen típicas adaptativas, también puede hacer que las imágenes se escalan para adaptarse a la pantalla que se utiliza para ver una página. Este es un gran ahorro de tiempo y otra buena razón para familiarizarse con el módulo.

Entendiendo la terminológia

A diferencia de algunos de los otros módulos de diseño, el módulo CSS Multi-column Layout utiliza un paradigma de diseño existente que todos conocemos. Esto hace que sea muy fácil de entender y empezar a utilizar.

Una columna es una división vertical en la que se representa el contenido, dentro de un contenedor global. Un contenedor puede contener una o más columnas, con el contenido fluyendo de uno a otro de acuerdo con las reglas del lenguaje que se está utilizando. Las columnas se procesan de izquierda a derecha. A medida que el contenido desborda la primera columna, comienza a visualizarse en la parte superior de la siguiente columna. Cuando se han utilizado todas las columnas, el contenido se desborda de acuerdo con las reglas normales de HTML / CSS, aunque esto puede dar lugar a columnas adicionales.

Una de las ventajas del módulo CSS Multi-column es que la altura del contenedor se calcula automáticamente para acomodar todo el contenido de forma predeterminada. Esto es útil y uno de los principales beneficios de utilizar el módulo sobre algunas de las soluciones más antiguas. Fuera de la caja, el contenido también se equilibra automáticamente en todas las columnas, lo que resulta en un agradable bloque de contenido delineado en columnas ordenadas para que coincida con su especificación.

Marcado HTML

No necesito hablar del HTML, porque se usa un elemento semántico estándar como contenedor del contenido. No hay requisitos especiales de marcado para utilizar el diseño de columnas múltiples CSS; Cualquier elemento de bloque puede tener aplicadas a él varias columnas utilizando este módulo.

Modelo Multi Columnas

Hay algunas cosas que debe advertirse acerca de cómo se procesa el contenido en el contenedor.
En el modelo de caja CSS original, el contenido de un elemento fluye en el cuadro de contenido de ese elemento.

CSS Multi-column Layout introduce un nuevo tipo de contenedor que existe entre el cuadro de contenido y el contenido real. Esto es referido por el W3C como el cuadro de columna (columna para abreviar). El contenido fluye automáticamente entre columnas dentro de un elemento, en la dirección en línea (o lectura). Esto es de izquierda a derecha en idiomas latinos como el español. El resultado del listado generado a 320 píxeles de ancho: la resolución de teléfono más común.

Las columnas se organizan en filas. Todas las columnas tienen una anchura y una altura comunes dentro de una fila. Las columnas pueden tener espacio entre ellas, que se conoce como la brecha de la columna (o canal al hablar de la impresión). En la mayoría de los casos en pantalla, un elemento dividido en columnas sólo tiene una fila. Sin embargo, hay casos especiales en los que un elemento puede contener varias filas de columnas. Los documentos impresos también pueden consistir en varias filas, donde un área de contenido ocupa más de una página impresa.

Utilizo el lenguaje del W3C al explicar la disposición de múltiples columnas CSS3 Layout. Por esa razón, me refiero al elemento multicol cuando se habla del contenedor que tiene las propiedades de varias columnas que se le aplican. Un elemento de varias columnas (multicol) es cualquier contenedor cuyo column-width (ancho de columna) o column-count (recuento de columnas) no se establece en auto. Como has visto, en este caso estoy usando un elemento <article>. Para evitar la confusión, también uso multicol como el nombre de la clase que pertenece a este elemento.

Todas las propiedades individuales cubiertas en la siguiente sección se aplican al elemento multicol. En CSS3, no es posible establecer propiedades y valores en columnas individuales. Para ser claro, esto significa que no se puede asignar fondo, relleno o margen para una columna específicamente.

El relleno padingy el margen margin aplicados a un elemento multicol son aplicados al contenedor, no a sus columnas

Aunque no se pueden controlar columnas individuales, es importante entender que el renderizador del navegador las trata como cajas separadas a nivel de bloque como las celdas de tabla. Cada cuadro de columna actúa como el bloque de contenido para su contenido. Esto significa, por ejemplo, que el margin y el padding aplicados a un elemento de párrafo representado en una columna se aplican contra el borde del bloque que contiene la columna.

Propiedades CSS

La propiedad columns es una abreviatura para definir valores para dos propiedades CSS

copy 

.multicol {  
    columns: 4 12em; 
}
column-count

column-count define el número de columnas en las que se debe dividir un elemento multicol. Las reglas determinan si el navegador respeta esta propiedad. Si no se especifica o se establece en auto, el espacio se divide según el valor de la propiedad width-column.

column-widt

column-width especifica el ancho mínimo que cada columna debe ocupar dentro del elemento multicol. Tenga en cuenta que he dicho mínimo, no definitivo! Una vez más, el motor de renderizado sigue una serie de reglas lógicas para determinar si esta regla se cumple estrictamente. Si no se especifica o se establece en auto, el ancho de columna se determina por el número de columnas en las que se divide, de acuerdo con el valor de la propiedad column-count.

columns

La propiedad columns es un método abreviado eficaz para la combinación de las propiedades column-count y column-width. Dependiendo de los valores que se pasan a la propiedad columns, el navegador interpreta sus intenciones de manera diferente, por lo que es importante comprender todas las permutaciones posibles. He tomado prestado el código de ejemplo del W3C para ayudar a ilustrarlo. Una manera simple de recordar cómo funciona esto es que si utiliza un valor único con una unidad, el navegador interpreta esto como un valor de ancho de columna. Si omite una unidad, el valor se interpretará como un valor de recuento de columnas. Si no está seguro, especifique dos valores: uno con una unidad para el width y uno sin unidad para count.

copy 

columns: 12em;      /* equates to column-width: 12em; column-count: auto */ 
columns: auto 12em; /* equates to column-width: 12em; column-count: auto */ 
columns: 2;         /* equates to column-width: auto; column-count: 2    */ 
columns: 2 auto;    /* equates to column-width: auto; column-count: 2    */ 
columns: auto;      /* equates to column-width: auto; column-count: auto */ 
columns: auto auto; /* equates to column-width: auto; column-count: auto */
Las reglas para eliminar columnas y cambiar anchos

Como se mencionó, el motor de renderizado del navegador no siempre se adhiere estrictamente a su definición de la anchura y el número total de columnas para un elemento multicol. Las reglas de cómo y cuándo cambiar la intención de renderizado están definidas por la especificación del módulo de diseño de columnas múltiples CSS3 del W3C. El W3C ofrece una lista de pseudocódigo para mostrar la lógica que se va a aplicar.

copy 

if ((column-width = auto) and (column-count = auto)) then     
    exit; /* not a multicol element */ 
    
if ((available-width = unknown) and (column-count = auto)) then     
    exit; /* no columns */ 
if (available-width = unknown) and (column-count != auto) and (column-width != auto) then     
    N := column-count;     
    W := column-width;     
    exit; 
if (available-width = unknown) then    
    available-width := shrink-to-fit; 
    
if (column-width = auto) and (column-count != auto) then     
    N := column-count;     
    W := max(0, (available-width - ((N - 1) * column-gap)) / N);     
    exit; 
    
if (column-width != auto) and (column-count = auto) then     
    N := max(1, floor((available-width + column-gap) / (column-width + column-gap)));     
    W := ((available-width + column-gap) / N) - column-gap;     
    exit; 
    
if (column-width != auto) and (column-count != auto) then     
    N := min(column-count, floor((available-width + column-gap) / (column-width + column-gap)))     
    W := ((available-width + column-gap) / N) - column-gap;     
    Exit

Las reglas y condiciones son comparadas y actuadas en orden:

  • Si las propiedades del column-count y del column-width están desactivadas o establecidas en auto, no se renderizan columnas. Lo mismo si column-count se establece en auto y el ancho del elemento multicol no está restringido.
  • Si no se han especificado las propiedades del column-count y del column-width tanto para los valores como para el elemento multicol, utilice los valores exactamente como se especifican.
  • Si la propiedad column-count se ha establecido con un valor distinto de auto y la propiedad column-width se ha establecido en auto, divida el ancho disponible en el número de columnas especificado.
  • Si la propiedad column-width se ha establecido con un valor distinto de auto, la propiedad column-count se ha establecido en auto y el contenedor tiene un ancho especificado, establezca el número de columnas para que coincida con el espacio disponible dividido por el ancho especificado de cada columna. Esto puede significar que las columnas terminan más allá de lo especificado, si la anchura del elemento multicol no se divide perfectamente en el ancho especificado de cada columna.
  • Si las propiedades column-count y column-width se han establecido con un valor distinto de auto y se ha establecido el ancho del elemento multicol, compruebe si la combinación de ancho y recuento se ajustará dentro del elemento de contenedor. Si no es así, reduzca el número de columnas para que coincida con el número de veces que el ancho especificado de cada columna puede caber en el espacio disponible y, a continuación, expanda el ancho de columna para llenar el elemento multicol utilizando el column-count calculado.

Este enfoque hace que el módulo CSS3 Multi-column Layout sea muy flexible. Está diseñado para responder automáticamente al intentar respetar los valores especificados para las propiedades column-count y column-width lo mejor posible.

column-gap

column-gap se utiliza para definir el ancho de canal entre columnas.

copy 

.multicol {   
    position:    relative;   
    margin:    auto;   
    max-width:    960px;   
    columns:    4 12em;   
    column-gap:    2em;   
    column-rule:    1px solid red; 
}

A diferencia de las propiedades column-width y column-count, la propiedad column-gap es absoluta. Esto significa que si especifica un valor, el explorador respetará esa configuración, aunque signifique sacrificar columnas.

column-rule

La propiedad column-rule controla si una línea divisoria se dibuja entre columnas renderizadas. Una column-rule siempre se dibuja en el centro exacto de la brecha de la columna (cuneta). column-rule funciona de la misma manera que la propiedad de border. De hecho, como con la propiedad border, column-rule es una propiedad abreviada para tres propiedades diferentes:

  • column-rule-width: Como con border-width, esta propiedad toma un argumento con una unidad como 2px o la palabra clave none.
  • column-rule-style: establece el estilo de línea que se va a mostrar. Las mismas opciones están disponibles como con borde-estilo, incluyendo punteado, punteado, sólido, y así sucesivamente.
  • column-rule-color: Este atributo de color se puede establecer utilizando cualquiera de los modelos de color admitidos por el navegador, incluidos hex y rgb ()

column-rule muestra como si la columna estuviera completamente llena de contenido, independientemente de si esto es cierto. Esto es útil si su definición de hoja de estilos requiere algo más que un relleno equilibrado en sus columnas, ya que ayuda a definir visualmente el espacio que ocupa una columna, incluso si está vacía de contenido. ¡Piense en lo difícil que fue lograr este efecto simple usando elementos flotantes para generar columnas!

Column Breaks

Normalmente, el motor de renderizado del navegador decide cómo y dónde romper el contenido entre columnas dentro de un elemento multicol. Esto funciona bien en muchos casos, pero puede haber momentos en los que desee controlar la forma en que se rompe el contenido. Por ejemplo, es posible que desee obligar al contenido a comenzar a representar en una nueva columna antes de cada etiqueta <h3>. Tres propiedades están disponibles para proporcionar control sobre cómo y cuándo romper el contenido: break-before, break-after y break-inside.

break-before

Esta propiedad especifica que debe aplicarse una ruptura antes del elemento al que se aplica. Acepta los siguientes valores: auto | always | avoid | left | right | page | column | avoid-page | avoid-column. De particular interés son las opciones column y avoid-column, que aseguran o impiden (donde sea posible) una ruptura en un elemento multicol.

break-after

Esta propiedad especifica que debe aplicarse una ruptura después del elemento al que se aplica. Acepta los siguientes valores: auto | always | avoid | left | right | page | column | avoid-page | avoid-column. De particular interés son las opciones column y avoid-column, que aseguran o impiden (donde sea posible) una ruptura en un elemento multicol.

break-inside

A diferencia de las otras dos opciones, break-inside controla cómo el contenido se rompe dentro del elemento al que se aplica. Acepta los siguientes valores: auto | always | avoid | avoid-page | avoid-column. De particular interés son las opciones column y avoid-column, que aseguran o impiden (donde sea posible) una ruptura en un elemento multicol.

Cuando una ruptura de columna divide una caja, el margen, el borde y el relleno de esa caja no tienen efecto visual sobre dónde se produce esa división. El margen inmediatamente después de la división se mantiene, sin embargo.

column-span

La propiedad column-span permite que un elemento de nivel de bloque dentro de una columna rompa fuera de los límites de esa columna. En el futuro, puede ser posible especificar un número de columnas para abarcar, al igual que el atributo colspan que se puede aplicar a una celda de tabla, pero en la especificación CSS3 sólo hay dos valores posibles: ninguno y todos

De forma predeterminada, column-span se establece en none. Esto significa que el contenido se renderiza dentro de las restricciones de la columna. Si se establece en all, la columna se rompe en el punto que se representa el elemento. Esto crea un efecto donde un elemento multicol puede contener varias filas de columnas.

Imágenes, capacidad de respuesta y recorte en columnas

Las imágenes presentan un problema especial en un diseño en el que no puede estar absolutamente seguro de cuánto espacio se asignará a su pantalla. Hay dos maneras de enfocar las imágenes en un diseño de columnas:

  • Llene el ancho de la columna con una imagen, ampliando o reduciendo la imagen para que coincida con el espacio disponible.
  • Predeterminar el tamaño en el que una imagen debe mostrarse, y envolver el texto alrededor o recortar la imagen como sea necesario para llenar el espacio.

El uso de una regla CSS que asigna una anchura al 100% obliga a la imagen a renderizar con el ancho completo de la columna. Esto efectivamente hace que la imagen responda, ya que crece y se encoge para que coincida con el espacio disponible.

Controlar cómo se llenan las columnas de contenido

Las columnas se pueden rellenar con contenido de dos maneras usando el diseño de columnas múltiples de CSS3: pueden tener contenido equilibrado entre ellas o se deben rellenar secuencialmente hasta que el contenido se agote. Si utiliza la última opción, existe la posibilidad de que una o más columnas se puedan renderizar sin ningún contenido.

La propiedad column-fill proporciona control sobre qué enfoque utiliza el navegador. Acepta dos valores posibles: balance y auto. Como es de esperar, el equilibrio hace que el contenido sea igual en todas las columnas disponibles, y el auto utiliza el enfoque secuencial. El valor predeterminado (no especificado) es el balance; por lo tanto, si no se establece explícitamente el relleno de columna en automático, el navegador intenta representar un relleno de columna equilibrado.

Cómo usar el Módulo de CSS multi-columnas

Como puede ver, CSS Multi-column Layout proporciona una solución sensible y práctica al problema de organizar el contenido en columnas similares a un periódico. Quizás el mayor beneficio del módulo es las mejoras en el flujo de trabajo que aporta al proceso de diseño web. Además, el módulo responde de forma predeterminada, ofreciendo una opción realmente útil para una metodología de diseño móvil-first y reduciendo la cantidad de tiempo de desarrollo necesario para lograr una solución eficaz en todas las resoluciones de pantalla.

CSS Multi-column Layout es una gran herramienta de diseño general, pero también se puede utilizar para crear elementos de interfaz de usuario más pequeños en la página, como estos:

  • Formularios en los que el contenido está organizado en columnas en pantallas más grandes, pero se reduce a una sola columna en dispositivos móviles
  • Menús horizontales de navegación en los que es importante que cada elemento ocupe la misma cantidad de espacio horizontal (aunque Flexbox obtiene una opción aún mejor para este propósito)
  • Renderización de contenido en áreas donde no se sabe de antemano cuánto contenido o espacio horizontal tendrá disponible

Dicho esto, el propósito principal de CSS Multi-column Layout es como una herramienta general de diseño de página, y aquí es donde obtendrá el máximo beneficio de ella.

Soporte del navegador

El soporte de navegador para CSS Multi-column Layout ya es muy bueno. Cada navegador principal ha tenido soporte para al menos dos versiones. Actualmente, Internet Explorer tiene la implementación más completa, con la versión actual ofreciendo soporte completo y soporte parcial para la especificación desde la versión 10. Firefox, Chrome, Safari y Opera también cuentan con soporte parcial de trabajo para la especificación, con Navegadores WebKit que ofrecen soporte completo. En las plataformas móviles, es similar: todos los navegadores comunes tienen un buen soporte, con IE móvil ofreciendo la implementación más completa de todos.

El módulo CSS Multi-column Layout alcanzó el estado de Recomendación Candidate en abril de 2011, lo que significa que es un módulo completo que puede considerar estable. A pesar de esto, el CSSWG sigue trabajando para refinar y desarrollar el módulo. Puede comprobar la compatibilidad con el navegador más reciente para CSS Multi-column Layout

No asuma que cada navegador renderizará su elemento multicol en forma idéntica.

Los navegadores que no entienden las propiedades de diseño de columnas múltiples de CSS vuelven a sus valores predeterminados para cada elemento. Esto significa que, en la mayoría de los casos, los elementos diseñados para que varias columnas se procesen en una sola columna. Para muchos diseños, esto no es una gran cuestión y es un resultado razonable para un enfoque de mejora progresiva en el diseño. Si es absolutamente necesario tener columnas en su diseño, independientemente del soporte del navegador para el diseño de columnas múltiples de CSS, el uso del marco Modernizr le permitirá identificar y manejar navegadores que no pueden procesar columnas CSS.

En este ejemplo, se utiliza el módulo CSS Multi-column Layout para representar todas las columnas que se muestran en el diseño, incluidas las columnas anidadas en la parte superior del diseño, donde hay tres menús en una fila. También utiliza varias de las características descritas en este capítulo, incluyendo la propiedad column-span para crear una pull-quote efectiva.

El código HTML para esta página es muy simple. En su núcleo, hay un conjunto de elementos, cada uno de los cuales contiene un solo artículo. El marcado funciona independientemente de si el navegador del usuario admite el diseño de varias columnas de CSS.
