Usar WebStorage API para la persistencia de datos de usuario

las aplicaciones web necesitan almacenar datos. Con LocalStorage de HTML5 y SessionStorage APIs se tiene una forma sencilla de alamcenar datos.

Lo básico

LocalStorage es un conjunto de pares nombre-valor, como con las cookies, pero a diferencia de estas los datos no se envían al servidor en cada solicitud. En su lugar JavaScript tiene acceso para leer, asignar o eliminar esos valores cuando sea necesario.

LocalStorage es almacenada por dominio. Es salvado a un archivo físico en la máquina del usuario. Esto tiene dos implicaciones. Primero, hay un límite a la cantidad total de datos que sepueden almacenar en LocalStorage. Segundo, leer y escribir es una operación simple de I/O.

LocalStorage es soportado por los navegadores de ordenadores y móviles.

API LocalStorage

Hay dos principales opciones para trabajar con LocalStorage:

Escribir y asignar valores usando directamene el acceso a window.

window.localStorage.favoriteBeer = "Abita";

window.localStorage["favoriteBeer"] = "Abita";

O utilizando los siguientes métodos:

  • setItem(key, value);
  • getItem(key);
  • removeItem(key);
  • clear();
